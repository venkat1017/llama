Autograd: Backpropagation
from Scratch

⏱ ~60 min 🧠 Core Mechanism 💻 CPU only

How does PyTorch "know" how to update weights? This lesson tears open the black box — we implement backprop by hand, derive the chain rule, and then watch PyTorch do it all automatically. This is the engine that trains every neural network.

The Preamble: Why Weights, Biases, and Calculus?

Before computing gradients, let's take a step back. If we have inputs (x) and targets (y), why do we even need weights (W) and biases (b)? And why do we use calculus and the chain rule instead of just using probability or solving it exactly?

Why W and b? (The Knobs)

Think of a neural network as a giant, complex soundboard. The input x is the raw audio. W (Weights) are the volume sliders—they decide how much "attention" to pay to each input. b (Biases) are the baseline offsets. Together, they are the learnable knobs that allow a rigid mathematical equation to morph, bend, and fit the shape of your data.

So, we have a billion knobs. How do we tune them to get the right output? Historically, scientists tried several ways:

  • Random Guessing (Brute Force): Try random knob combinations. This works for 2 knobs, but is mathematically impossible for 8 billion knobs (like Llama). The universe would end before you found a good combination.
  • Analytical Math (Closed-form): For simple lines, you can use algebra to jump straight to the exact perfect answer. But add a non-linear "kink" (like a ReLU valve), and the exact math becomes impossible to solve backwards in one go.
  • Evolution/Probability: Make random mutations, see what survives. Better, but very slow because it's essentially "blind" trial and error. It doesn't tell you which direction to move.
🧭
Calculus gives us a Compass. Unlike blind guessing, Calculus allows us to ask a highly specific question: "If I turn this exact knob slightly to the right, does the error go up or down?" Because neural networks are built of sequential layers, the Chain Rule is the only mathematical mechanism that lets us pass this "compass direction" backwards from the final output all the way to the very first input.

The Big Picture: Why Do We Need Gradients?

Training a neural network is an optimization problem: we want to find the values of our weights that minimize the loss (how wrong our predictions are). To do that, we need to know the answer to the calculus question: "If I change weight W by a tiny amount, how much does the loss change?"

That rate of change is the gradient. Gradients point uphill (toward higher loss), so we move the weights downhill — this is gradient descent.

The Chain Rule — in one sentence

If loss depends on y, and y depends on x, then how much does loss change when x changes? Answer: (∂loss/∂y) × (∂y/∂x). The chain rule lets us decompose complex gradients into simple multiplications — and that's exactly what backprop does.

Implement Backprop by Hand

We'll build the simplest possible neural network: one linear layer + MSE loss. We'll compute gradients manually first, then verify with PyTorch's autograd. This is the same thing Karpathy does in micrograd.

manual_backprop.py — no PyTorch autograd
MANUAL
import torch

# ── Our tiny model: y = w*x + b ───────────────────────────────
# We'll compute ALL gradients ourselves, step by step.

x      = 3.0   # input (fixed, not learned)
w      = 2.0   # learnable weight
b      = 1.0   # learnable bias
y_true = 5.0   # target

# ── FORWARD PASS ─────────────────────────────────────────────
u      = w * x           # u = 6.0
y_pred = u + b           # y_pred = 7.0
loss   = (y_pred - y_true) ** 2  # loss = 4.0
print(f"Forward: u={u}, y_pred={y_pred}, loss={loss}")

# ── BACKWARD PASS (chain rule by hand) ───────────────────────
# We go backwards through each operation.

# Step 1: gradient of loss w.r.t. loss = 1 (trivially)
d_loss__d_loss = 1.0

# Step 2: loss = (y_pred - y_true)²
# d(loss)/d(y_pred) = 2*(y_pred - y_true)
d_loss__d_ypred = 2 * (y_pred - y_true)   # = 2*(7-5) = 4.0

# Step 3: y_pred = u + b
# d(y_pred)/d(u) = 1  → so d_loss/d_u = d_loss/d_ypred * 1
d_loss__d_u = d_loss__d_ypred * 1.0          # = 4.0

# d(y_pred)/d(b) = 1  → so d_loss/d_b = d_loss/d_ypred * 1
d_loss__d_b = d_loss__d_ypred * 1.0          # = 4.0

# Step 4: u = w * x
# d(u)/d(w) = x   → so d_loss/d_w = d_loss/d_u * x
d_loss__d_w = d_loss__d_u * x               # = 4.0 * 3.0 = 12.0

print(f"∂loss/∂w = {d_loss__d_w}")   # 12.0
print(f"∂loss/∂b = {d_loss__d_b}")   # 4.0

# ── VERIFY: PyTorch computes the same thing automatically ─────
w_t = torch.tensor([2.0], requires_grad=True)
b_t = torch.tensor([1.0], requires_grad=True)
x_t = torch.tensor([3.0])  # no grad needed for input

y_pred_t = w_t * x_t + b_t
loss_t   = (y_pred_t - 5.0) ** 2
loss_t.backward()

print(f"PyTorch ∂loss/∂w = {w_t.grad.item()}")  # 12.0 ✓
print(f"PyTorch ∂loss/∂b = {b_t.grad.item()}")  # 4.0  ✓
✓
Identical results. Our manual calculation and PyTorch's autograd produce exactly the same gradients. PyTorch just does this automatically for every operation in your network — no matter how deep or complex. That's the power of the computation graph.

The Computation Graph in Depth

When you write y = w * x + b with requires_grad=True, PyTorch secretly builds a graph of Function nodes — one for each operation. Each node knows how to compute its local gradient (the derivative of its output w.r.t. its inputs).

When you call loss.backward(), PyTorch traverses this graph in reverse order (from loss back to weights), multiplying local gradients together — that's the chain rule in action.

inspect_graph.py — peek inside autograd
INTERNALS
import torch

w = torch.tensor([2.0], requires_grad=True)
x = torch.tensor([3.0])
b = torch.tensor([1.0], requires_grad=True)

y = w * x + b
loss = (y - 5.0) ** 2

# PyTorch attaches a .grad_fn to every tensor created from learnable params
print(loss.grad_fn)        # PowBackward0
print(y.grad_fn)           # AddBackward0
print((w*x).grad_fn)      # MulBackward0

# Each grad_fn links to its inputs — that's the graph
print(loss.grad_fn.next_functions)
# → ((SubBackward0,...), (None,...))
# It knows: to get grad of loss, first go through Sub, then Pow

# ── The gradient accumulation trap ───────────────────────────
# IMPORTANT: .backward() ACCUMULATES gradients by default!
# If you call it twice without zeroing, grads stack up.

w.grad = None  # reset
loss.backward()
print(w.grad)   # tensor([12.])

# Oops: backward again WITHOUT zeroing
loss2 = (w * x + b - 5.0) ** 2
loss2.backward()
print(w.grad)   # tensor([24.]) ← WRONG! doubled because of accumulation

# Fix: always zero gradients before each backward pass!
# In the training loop you'll do:  optimizer.zero_grad()
⚠️
The #1 training bug: forgetting to zero gradients. Always call optimizer.zero_grad() (or model.zero_grad()) at the start of each training step. If you don't, gradients from previous batches accumulate and your model trains on garbage.

A Real Mini-Network: The Matrix Dimension Trick

Going from scalars (single numbers) to matrices (grids of numbers) is where 90% of people get stuck. But here is the secret "cheat code" used by neural network engineers: The shape of a gradient must exactly match the shape of its original tensor.

The Local Node Mental Model

Don't look at the whole network at once. Think of every operation (like a matrix multiply or a ReLU) as an isolated "Node" sitting in the graph. During the backward pass, a Node receives a gradient from above (d_out). Its only job is to calculate: "How do I route this gradient to my inputs?"

Let's look at a 2-layer network with a ReLU activation. This is the exact mathematical structure inside every Transformer feed-forward block.

Forward: h = ReLU(W₁·x + b₁), y = W₂·h + b₂
Deriving the backward pass step-by-step:

1. The Linear Layer (y = W₂·h): How do we find d_W2?
We know W₂ has shape (d_out, d_hidden). We receive d_y from above with shape (d_out, 1). We have the input h with shape (d_hidden, 1).
How do we combine (d_out, 1) and (d_hidden, 1) to get (d_out, d_hidden)?
Answer: The Outer Product! d_y @ h.T

2. The ReLU Valve (h = ReLU(z)): How do we find d_z?
ReLU is a literal physical valve. If the forward pass z was > 0, the valve is OPEN (derivative is 1, gradient flows perfectly). If z was ≤ 0, the valve is CLOSED (derivative is 0, gradient is permanently destroyed).
mlp_manual.py — 2-layer net, manual backprop
DEEP DIVE
import torch

torch.manual_seed(42)

# ── Network hyperparameters ───────────────────────────────────
d_in, d_hidden, d_out = 4, 8, 2

W1 = torch.randn(d_hidden, d_in)  * 0.1
b1 = torch.zeros(d_hidden)
W2 = torch.randn(d_out, d_hidden) * 0.1
b2 = torch.zeros(d_out)

x      = torch.randn(d_in)         
target = torch.tensor([1.0, 0.0])

# ── FORWARD PASS ─────────────────────────────────────────────
z1 = W1 @ x + b1              # Shape: (8,)
h  = torch.relu(z1)            # Shape: (8,)
z2 = W2 @ h + b2              # Shape: (2,)
loss = ((z2 - target) ** 2).mean()

# ── BACKWARD PASS ────────────────────────────────────────────
# 1. Gradient of MSE Loss: d_loss/d_z2
d_z2 = 2.0 * (z2 - target) / d_out  # Shape: (2,)

# 2. Backprop through Layer 2 (y = W2 @ h + b2)
# SHAPE MATCHING: d_W2 must be (2, 8). 
# d_z2 is (2,), h is (8,). Outer product gives us (2, 8)!
d_W2 = torch.outer(d_z2, h)       
d_b2 = d_z2.clone()               

# To continue backward, we need the gradient w.r.t the input 'h'
# SHAPE MATCHING: d_h must be (8,). W2 is (2, 8), d_z2 is (2,).
# W2.T @ d_z2 -> (8, 2) @ (2,) = (8,)
d_h  = W2.T @ d_z2               

# 3. Backprop through ReLU (The Valve)
# If z1 > 0, pass d_h backward. If z1 <= 0, multiply by 0.
d_z1 = d_h * (z1 > 0).float()   

# 4. Backprop through Layer 1
# SHAPE MATCHING: d_W1 must be (8, 4). d_z1 is (8,), x is (4,)
d_W1 = torch.outer(d_z1, x)       
d_b1 = d_z1.clone()               

# Check shapes to ensure sanity!
print(f"W1 shape: {W1.shape} == d_W1 shape: {d_W1.shape}")  # (8, 4) == (8, 4)
print(f"W2 shape: {W2.shape} == d_W2 shape: {d_W2.shape}")  # (2, 8) == (2, 8)

# ── VERIFY WITH AUTOGRAD ──────────────────────────────────────
W1_ag = W1.clone().requires_grad_(True)
W2_ag = W2.clone().requires_grad_(True)
b1_ag = b1.clone().requires_grad_(True)
b2_ag = b2.clone().requires_grad_(True)

z1_ag = W1_ag @ x + b1_ag
h_ag  = torch.relu(z1_ag)
z2_ag = W2_ag @ h_ag + b2_ag
loss_ag = ((z2_ag - target) ** 2).mean()
loss_ag.backward()

# Check they match!
print(torch.allclose(d_W1, W1_ag.grad))  # True ✓
print(torch.allclose(d_W2, W2_ag.grad))  # True ✓
💡
Think of Gradients as "Tug of War" Forces. When we compute d_loss__d_w = 12.0, what does that actually mean? It means the loss function is "pulling" on the weight w with a force of +12.0. If we increase w, the loss goes UP. To train the network, we simply step in the opposite direction of the force (w = w - 0.01 * 12.0). Backpropagation is just computing these pulling forces for every single weight in the network simultaneously.

torch.no_grad() — Disabling the Graph

When you're doing inference (generating text, evaluating the model), you don't need gradients. Building the computation graph wastes memory and time. Use torch.no_grad() to disable it:

no_grad_usage.py
IMPORTANT
import torch

model_weight = torch.randn(4, 4, requires_grad=True)
x = torch.randn(4)

# During training: grad_fn is tracked
y_train = model_weight @ x
print(y_train.grad_fn)   # MmBackward0 — graph is built

# During inference / evaluation: no graph needed
with torch.no_grad():
    y_infer = model_weight @ x
    print(y_infer.grad_fn)  # None — no graph, less memory!

# You'll use this pattern constantly in Lesson 14 (inference)
# Also use @torch.inference_mode() decorator for the same effect

  2. The Matrix Cheat Code: You have a weight matrix W of shape (64, 128). You don't know the forward math, but you need to write the backward pass in PyTorch. What will be the exact shape of the gradient tensor d_W?

    loss.backward()
    optimizer.step()
    What single critical line of code is missing, and why did its absence cause the network to explode?
