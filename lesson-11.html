The Training Loop

⏱ ~70 min🔁 Optimization💻 CPU only

The model exists. Now we make it learn. The training loop is a tight four-step cycle repeated thousands of times: forward pass, compute loss, backward pass, update weights. Every LLM ever trained runs this same loop.

The Four-Step Cycle

Each "step" processes one batch of token sequences. After thousands of steps, the model's weights converge to values that minimize loss on the training data.

1
Forward Pass
Run batch through model → logits [B, S, vocab_size]
logits = model(x)
2
Compute Loss
Cross-entropy vs correct next tokens. Random ≈ 5.5. Good ≈ 1.5.
loss = F.cross_entropy(logits.view(-1,V), y.view(-1))
3
Backward Pass
Compute gradient of loss w.r.t. every parameter. Zero first!
optimizer.zero_grad()  # clear old grads
loss.backward()        # compute new grads
4
Update Weights
Optimizer nudges each weight to reduce loss. Clip grads first.
clip_grad_norm_(model.parameters(), 1.0)
optimizer.step()
Why zero_grad() before backward()?

PyTorch accumulates gradients into .grad by default. Without zeroing, each backward() adds onto the previous batch's gradients - you'd train on a blend of all past batches. Always: zero → backward → step, every single iteration.

Cross-Entropy Loss for LMs

At every sequence position, the model predicts which token comes next. Cross-entropy measures: how much probability did we assign to the correct token? loss = -log(prob_correct). If uniform random over 256 tokens: -log(1/256) = 5.54. Good trained model: under 1.5.

loss_mechanics.py
CONCEPT
import torch
import torch.nn.functional as F

B, S, V = 4, 64, 256
logits  = torch.randn(B, S, V)                  # model output
targets = torch.randint(0, V, (B, S))           # correct next tokens

# F.cross_entropy needs: inputs (N, C), targets (N,)
# Flatten batch and sequence dims into N = B*S
loss = F.cross_entropy(logits.view(-1, V), targets.view(-1))
print(f"Loss: {loss.item():.3f}")      # ~5.5 - random baseline
print(f"PPL:  {loss.exp().item():.1f}")  # ~256 - model "unsure" among all tokens

# Manual verification of what cross_entropy does internally:
probs  = F.softmax(logits[0, 0], dim=-1)
manual = -torch.log(probs[targets[0, 0]])
print(f"Manual (pos 0): {manual.item():.3f}")  # matches!

# DataLoader convention - shifted by one position:
#   x = tokens[i   : i+seq_len]   ← input
#   y = tokens[i+1 : i+seq_len+1] ← targets (the "next tokens")
# At position j, model uses x[j] to predict y[j] = x[j+1]

Complete Training Script

train.py - full loop with validation and checkpointing
CORE CODE
import torch, time
import torch.nn.functional as F

def train_epoch(model, loader, optimizer, device='cpu'):
    model.train()   # enable training mode
    total, n = 0.0, 0
    for step, (x, y) in enumerate(loader):
        t0 = time.time()
        x, y = x.to(device), y.to(device)
        B, S, _ = (logits := model(x)).shape
        loss = F.cross_entropy(logits.view(B*S, -1), y.view(B*S))
        optimizer.zero_grad()
        loss.backward()
        gnorm = torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        optimizer.step()
        total += loss.item(); n += 1
        if step % 100 == 0:
            tok_s = B*S / (time.time()-t0)
            print(f"  step {step:4d} loss={loss.item():.3f} gnorm={gnorm:.2f} tok/s={tok_s:.0f}")
    return total / n

def evaluate(model, loader, device='cpu'):
    model.eval()   # disable dropout / training-only behaviour
    total, n = 0.0, 0
    with torch.no_grad():   # skip building computation graph
        for x, y in loader:
            x, y = x.to(device), y.to(device)
            B, S, _ = (logits := model(x)).shape
            total += F.cross_entropy(logits.view(B*S,-1), y.view(B*S)).item()
            n += 1
    return total / n

# ── Main ─────────────────────────────────────────────────────
cfg   = NanoLlamaConfig()
model = NanoLlama(cfg)
opt   = torch.optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.1)

best_val = float('inf')
for epoch in range(10):
    tr = train_epoch(model, train_loader, opt)
    vl = evaluate(model, val_loader)
    print(f"Epoch {epoch+1} | train={tr:.3f} val={vl:.3f} ppl={vl.exp() if hasattr(vl,'exp') else __import__('math').exp(vl):.1f}")
    if vl < best_val:
        best_val = vl
        torch.save({'model': model.state_dict(), 'cfg': cfg}, 'best.pt')
        print("  ✓ checkpoint saved")
💡
model.train() vs model.eval(): Always switch mode before each phase. With torch.no_grad() during eval, PyTorch skips building the computation graph - roughly 2× faster and uses much less memory. Never run eval without both.

What You Now Know

  • The 4-step cycle: forward → loss → zero+backward → clip+step
  • Cross-entropy: -log(prob of correct token). Random baseline ≈ 5.54
  • zero_grad() before every backward() - non-negotiable
  • clip_grad_norm_ prevents rare exploding gradient catastrophes
  • train() / eval() + no_grad() for correct evaluation behaviour
