About the Course

Philosophy

This course exists because most LLM tutorials use high-level wrappers that hide the actual math. nanoLlama is a deliberate constraint: pure PyTorch, no HuggingFace, no einops. Every operation written explicitly.

The model is intentionally tiny (215K params) so it trains in under 30 minutes on a laptop CPU. The point is never the result — it's understanding every computation that produces it.

Prerequisites
Python — comfortable with classes and list comprehensions
Basic calculus — derivatives, chain rule (we re-derive in Lesson 02)
NumPy or PyTorch — basic tensor operations
No ML background required — we build from first principles
Tech Stack
torch — tensor ops, autograd, nn.Module
torch.utils.data — DataLoader, Dataset
Python stdlib only — no external dependencies
Runs on CPU — any machine, no GPU required
What You'll Build
Byte-level tokenizer and text dataset pipeline
Multi-head attention with causal masking
SwiGLU feed-forward block and RMSNorm
RoPE positional encoding (same as Llama-3)
Complete transformer model, training loop, text generation
The nanoLlama Model
nanollama.py — complete model, ~100 lines PYTHON 
import torch
import torch.nn as nn

class NanoLlama(nn.Module):
    def __init__(self, vocab_size=256, d_model=64, n_heads=4, n_layers=4):
        super().__init__()
        self.embed  = nn.Embedding(vocab_size, d_model)  # 16,384 params
        self.blocks = nn.ModuleList([TransformerBlock(d_model, n_heads) for _ in range(n_layers)])
        self.norm   = RMSNorm(d_model)
        # Weight tying: output projection shares weights with embedding
        self.output = lambda x: x @ self.embed.weight.T  # 0 extra params

    def forward(self, tokens):            # [batch, seq_len]
        x = self.embed(tokens)             # → [batch, seq_len, 64]
        for block in self.blocks:
            x = block(x)                  # → [batch, seq_len, 64]
        return self.output(self.norm(x))  # → [batch, seq_len, 256]
