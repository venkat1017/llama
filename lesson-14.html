& Inference

The model is trained. Now we make it write. Inference is an autoregressive loop: feed a prompt, get a next-token distribution, sample from it, append the token, repeat. We implement greedy, temperature, and top-k sampling - each producing very different text.

The Autoregressive Generation Loop

Language models don't generate whole sentences at once. They generate one token at a time, feeding each new token back as input for the next step. This is "autoregressive" - each output depends on all previous outputs.

generation_loop.py - the core inference loop
CORE CODE
import torch
import torch.nn.functional as F

def generate(
    model,
    prompt_ids:   torch.Tensor,   # [1, prompt_len] - encoded prompt
    max_new_tokens: int = 200,
    temperature:  float = 0.8,
    top_k:        int = 40,
) -> torch.Tensor:
    """
    Generate tokens autoregressively.

    temperature=0.0: greedy (always pick highest-prob token)
    temperature=1.0: sample from raw distribution
    temperature>1.0: more random  |  <1.0: more conservative
    top_k: restrict sampling to top-k tokens only
    """
    model.eval()
    generated = prompt_ids.clone()  # start with prompt

    with torch.no_grad():
        for _ in range(max_new_tokens):
            # Truncate to max_seq_len (context window)
            ctx = generated[:, -model.cfg.max_seq_len:]

            # Forward pass - we only care about the LAST position's logits
            logits = model(ctx)            # [1, seq_len, vocab]
            logits = logits[:, -1, :]    # [1, vocab] ← last position only

            # ── Sampling strategy ────────────────────────────────
            if temperature == 0.0:
                # Greedy: always pick most likely token
                next_id = logits.argmax(dim=-1, keepdim=True)
            else:
                # Temperature: scale logits before softmax
                # Low T → sharper distribution → more predictable
                # High T → flatter distribution → more creative/random
                logits = logits / temperature

                # Top-k: zero out all but top k logits
                if top_k > 0:
                    top_vals, _ = torch.topk(logits, top_k, dim=-1)
                    threshold   = top_vals[:, -1].unsqueeze(1)
                    logits = logits.masked_fill(logits < threshold, float('-inf'))

                # Sample from the filtered distribution
                probs   = F.softmax(logits, dim=-1)
                next_id = torch.multinomial(probs, num_samples=1)

            # Append to sequence and continue
            generated = torch.cat([generated, next_id], dim=1)

    return generated

Temperature & Top-k: What They Do

sampling_effects.py - compare strategies on a fixed state
INSIGHT
import torch
import torch.nn.functional as F

# Suppose the model's top logits for the next token after "To be or not":
logits = torch.tensor([4.2, 3.1, 2.8, 1.0, -0.5])  # top 5 tokens
tokens = ["to", "not", "be", "or", "that"]

# T=0.0 (greedy) - always picks "to"
print("Greedy:", tokens[logits.argmax().item()])  # "to"

# T=1.0 (raw) - sample from original distribution
probs_raw = F.softmax(logits / 1.0, dim=-1)
print("T=1.0 probs:", [f"{p:.2f}" for p in probs_raw.tolist()])
# ['0.46', '0.17', '0.12', '0.10', '0.06']  ← "to" likely but not certain

# T=0.5 (sharper) - more confident, less random
probs_low = F.softmax(logits / 0.5, dim=-1)
print("T=0.5 probs:", [f"{p:.2f}" for p in probs_low.tolist()])
# ['0.72', '0.12', 0.08', '0.05', '0.02']  ← much more peaked

# T=2.0 (flatter) - more creative/random
probs_high = F.softmax(logits / 2.0, dim=-1)
print("T=2.0 probs:", [f"{p:.2f}" for p in probs_high.tolist()])
# ['0.29', '0.22', '0.20', '0.15', '0.10']  ← much flatter

# Top-k = 3: only sample from "to", "not", "be" - ignore rest
top3_vals, top3_idx = torch.topk(logits, 3)
print("Top-3 tokens:", [tokens[i] for i in top3_idx.tolist()])  # ["to","not","be"]
💡
Good defaults for nanoLlama generation: temperature=0.8, top_k=40. Lower temperature → more coherent but repetitive. Higher → more surprising but incoherent. Greedy generation often loops: "the the the the..." - temperature + top_k breaks repetition loops naturally.

Full Inference Script

infer.py - load checkpoint and generate text
COMPLETE
import torch
from nanollama import NanoLlama, NanoLlamaConfig

# Load trained model
checkpoint = torch.load('best.pt', map_location='cpu')
cfg   = checkpoint['cfg']
model = NanoLlama(cfg)
model.load_state_dict(checkpoint['model'])
model.eval()

# Byte tokenizer (from Lesson 03)
class ByteTokenizer:
    def encode(self, text): return list(text.encode('utf-8'))
    def decode(self, ids):  return bytes(ids).decode('utf-8', errors='replace')

tokenizer = ByteTokenizer()

# Generate text from a prompt
prompt = "HAMLET:
"
ids    = torch.tensor([tokenizer.encode(prompt)], dtype=torch.long)

output_ids = generate(
    model, ids,
    max_new_tokens=300,
    temperature=0.8,
    top_k=40,
)

# Decode only the new tokens (skip the prompt)
new_tokens = output_ids[0, len(ids[0]):].tolist()
print(prompt + tokenizer.decode(new_tokens))

# Expected output after ~10 epochs on TinyShakespeare:
#
# HAMLET:
# What means this, lord? I think we must speak
# of this same thought to be the heart that loves thee,
# And in the night, when all the world is done,
# The duke is in the chamber of the king...
#
# Not perfect - but recognisably Shakespeare-like! The model
# has learned: character names, blank verse rhythm, archaic
# vocabulary, and dialogue structure. Not bad for 215K params.

What You Now Know

  • Autoregressive generation: one token at a time, each fed back as input
  • Greedy (T=0): deterministic, often repetitive. Use temperature to add variety.
  • Temperature scales logits before softmax - lower = sharper, higher = flatter
  • Top-k restricts sampling to the k most likely tokens - prevents wild choices
  • Load checkpoints with torch.load + model.load_state_dict
