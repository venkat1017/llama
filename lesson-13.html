CPU Training:
Tips & Profiling

⏱ ~50 min⚡ Performance💻 CPU only

Training on CPU is slow - but not as slow as you might think if you profile carefully and eliminate bottlenecks. We'll measure where time actually goes, enable key PyTorch optimisations, and set up a training run that finishes in under 30 minutes on a modern laptop.

Where Does Time Go?

Before optimising, measure. The biggest time sinks in CPU LLM training:

profile_training.py - find your actual bottleneck
PROFILING
import torch, time

# Simple timing of each step component
def time_training_step(model, x, y, optimizer):
    timings = {}

    t0 = time.perf_counter()
    logits = model(x)
    timings['forward'] = time.perf_counter() - t0

    t0 = time.perf_counter()
    B, S, V = logits.shape
    loss = F.cross_entropy(logits.view(B*S,V), y.view(B*S))
    timings['loss'] = time.perf_counter() - t0

    t0 = time.perf_counter()
    optimizer.zero_grad(); loss.backward()
    timings['backward'] = time.perf_counter() - t0

    t0 = time.perf_counter()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
    timings['step'] = time.perf_counter() - t0

    total = sum(timings.values())
    for k, v in timings.items():
        print(f"  {k:10s}: {v*1000:6.1f}ms  ({v/total*100:.0f}%)")
    print(f"  {'total':10s}: {total*1000:.1f}ms")

# Typical result on Apple M2:
#   forward  :  85.3ms  (45%)   ← matrix multiplications (expected)
#   loss     :   2.1ms  ( 1%)
#   backward : 142.6ms  (76%)   ← backward always ~2x forward (also expected)
#   step     :  12.4ms  ( 7%)
# Total: ~190ms per step = ~5 steps/sec with batch=4, seq=64

Key CPU Optimisations

cpu_optimisations.py
PERFORMANCE
import torch

# 1. Thread count - default often too high on laptops
# Too many threads → context switching overhead
torch.set_num_threads(4)   # try 2, 4, 6 - benchmark each

# 2. BF16 for forward pass - 2x faster matmuls on modern CPUs
# BF16 = Brain Float 16 - same exponent range as FP32, less precision
# Supported natively on Intel Ice Lake+, Apple M1+, AMD Zen4+
with torch.autocast(device_type='cpu', dtype=torch.bfloat16):
    logits = model(x)   # runs matmuls in BF16 automatically
# Note: only forward pass benefits; backward stays FP32

# 3. torch.compile - traces the model and compiles to optimised native code
# One-time compilation cost (~60s), then 20-40% speedup on CPU
model_compiled = torch.compile(model, backend='inductor')
# Use model_compiled in the training loop instead of model
# Note: requires PyTorch 2.0+

Debugging tip: Run 10 steps first. If loss doesn't drop below 5.0 after 10 steps: check that targets are shifted by 1 (x = tokens[:-1], y = tokens[1:]). If loss is NaN from step 1: check for empty sequences, bad LR, or division by zero in RMSNorm eps.

What You Now Know

  • Profile first - backward is always ~2x forward, that's expected
  • torch.set_num_threads(4) often faster than default on laptops
  • torch.autocast('cpu', bfloat16) gives 20-40% speedup on modern CPUs
  • torch.compile adds one-time cost but speeds up each step significantly
  • Gradient accumulation simulates larger batches without extra memory
