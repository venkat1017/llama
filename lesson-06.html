Multi-Head
Attention

⏰ ~70 min 🔀 Parallelism 💻 CPU only

Single-head attention can only look at tokens from one perspective at a time. Multi-head attention runs several heads in parallel, each learning to focus on different relationships — syntax, coreference, semantics. Then it merges all perspectives.

Why Multiple Heads?

Consider understanding: "The animal didn't cross the street because it was too tired." To understand "it" refers to "animal" you need coreference tracking. But also syntax (subject-verb) and semantics (what "tired" implies). One head learns one kind of attention pattern. Multiple heads learn multiple patterns simultaneously.

With n_heads heads, each head operates on a smaller subspace: d_head = d_model / n_heads. Their outputs are concatenated and projected back to d_model. Total computation cost is identical to one big head — just split differently.

Key constraint: n_heads x d_head = d_model. Our nanoLlama: d_model=64, n_heads=4, d_head=16. The assert in __init__ enforces this. Never skip it.

The Reshape Trick — Efficient Parallel Heads

Naive implementation: loop over heads one by one (slow). Efficient implementation: reshape tensors so all heads run as a single batched matrix multiply. This is the trick every production transformer uses.

reshape_trick.py
KEY INSIGHT
import torch

B, S, D   = 2, 8, 64
n_heads   = 4
d_head    = D // n_heads   # = 16

# After the QKV projection we have Q of shape [B, S, D]
Q_full = torch.randn(B, S, D)

# Step 1: view as (B, S, n_heads, d_head)
Q_reshaped = Q_full.view(B, S, n_heads, d_head)
print(Q_reshaped.shape)   # [2, 8, 4, 16]

# Step 2: transpose so heads sit in dim-1 (treated as batch dimension)
# We want (B, n_heads, S, d_head) so matmul sees (n_heads, S, d_head)
Q_heads = Q_reshaped.transpose(1, 2)
print(Q_heads.shape)      # [2, 4, 8, 16] = (B, heads, S, d_head)

# PyTorch batched matmul now treats (B, heads) as independent batch dims.
# Q @ K.T: (B, H, S, d_h) @ (B, H, d_h, S) -> (B, H, S, S)
# All 4 heads computed at once — no loop needed!
K_heads = torch.randn(B, n_heads, S, d_head)
scores  = Q_heads @ K_heads.transpose(-2, -1)
print(scores.shape)       # [2, 4, 8, 8] = (B, heads, S, S)

# After attention, reverse the reshape:
# (B, H, S, d_h) -> transpose -> (B, S, H, d_h) -> view -> (B, S, D)
out = torch.randn(B, n_heads, S, d_head)
out = out.transpose(1, 2).contiguous().view(B, S, D)
print(out.shape)           # [2, 8, 64] = (B, S, D) — heads merged!

# .contiguous() is REQUIRED before .view() after .transpose()
# Transpose changes stride metadata but not memory layout;
# .view() needs contiguous memory — always add it.
⚠
The .contiguous() trap. After .transpose(), tensor memory is non-contiguous. .view() requires contiguous layout. Always call .contiguous() before .view() after a transpose. Alternatively use .reshape() which handles this automatically (but may copy data).

Complete Multi-Head Attention Module

multi_head_attention.py
FINAL CODE
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    """Causal multi-head self-attention. nanoLlama: d_model=64, n_heads=4."""

    def __init__(self, d_model: int, n_heads: int):
        super().__init__()
        assert d_model % n_heads == 0

        self.d_model = d_model
        self.n_heads = n_heads
        self.d_head  = d_model // n_heads   # 64//4 = 16
        self.scale   = math.sqrt(self.d_head)

        # One combined QKV projection — 3x more efficient than 3 separate layers
        self.qkv_proj = nn.Linear(d_model, 3 * d_model, bias=False)
        self.out_proj = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B, S, D = x.shape

        # 1. Project: (B,S,D) -> (B,S,3D) -> split into three (B,S,D)
        q, k, v = self.qkv_proj(x).chunk(3, dim=-1)

        # 2. Reshape for multi-head: (B,S,D) -> (B,H,S,d_h)
        def split_heads(t):
            return t.view(B, S, self.n_heads, self.d_head).transpose(1, 2)

        q, k, v = split_heads(q), split_heads(k), split_heads(v)
        # q,k,v: (B, 4, S, 16)

        # 3. Scaled dot-product attention (all heads batched)
        scores = (q @ k.transpose(-2, -1)) / self.scale   # (B, 4, S, S)

        mask   = torch.triu(torch.ones(S, S, device=x.device, dtype=torch.bool), diagonal=1)
        scores = scores.masked_fill(mask, float('-inf'))
        attn   = F.softmax(scores, dim=-1)          # (B, 4, S, S)
        out    = attn @ v                              # (B, 4, S, 16)

        # 4. Merge heads: (B,H,S,d_h) -> (B,S,D)
        out = out.transpose(1, 2).contiguous().view(B, S, D)

        # 5. Output projection
        return self.out_proj(out)   # (B, S, 64)

# --- Test ---
mha = MultiHeadAttention(d_model=64, n_heads=4)
x   = torch.randn(4, 64, 64)
out = mha(x)
print(out.shape)   # torch.Size([4, 64, 64])

params = sum(p.numel() for p in mha.parameters())
print(f"MHA params: {params:,}")   # 16,384

✓
scores is [B, H, S, S] — we have H independent attention matrices, one per head. Each head puts its attention mass in different places. Head 1 might attend locally; head 3 might attend to distant tokens. The model learns this freely.

What You Now Know

  • Multi-head runs H heads in parallel, each on d_head = d_model/H dims
  • Reshape trick: treat heads as a batch dim for one big batched matmul
  • Combined QKV projection more efficient than 3 separate Linears
  • .contiguous() required before .view() after .transpose()
  • MHA is shape-preserving: [B, S, D] in, [B, S, D] out
